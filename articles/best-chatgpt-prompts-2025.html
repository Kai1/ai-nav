🧱 提示词结构公式

好的提示词不靠运气，靠结构。以下是最常用的两个框架：

📐 框架一：ROLE + TASK + FORMAT

角色（Role）：你扮演一个[专业角色]

任务（Task）：请帮我[具体任务]，背景是[上下文]

格式（Format）：输出格式要求[长度/结构/风格]

📐 框架二：CONTEXT + OBJECTIVE + CONSTRAINT

背景（Context）：我是[身份]，目前[情况]

目标（Objective）：我需要[具体目标]

限制（Constraint）：要求[格式/长度/语气/不要做什么]

✍️ 写作 & 文案（10个）

📝 提示词 #1 — 文章大纲生成
你是一位专业内容策略师。请为我生成一篇关于"[话题]"的深度文章大纲。目标读者是[描述]，文章长度约[字数]字，风格[专业/轻松/教程式]。大纲需要包含：引人注意的开头钩子、3-5个有数据支撑的核心论点、实用的可操作建议、有力的结尾号召。每个段落请附一句内容提示。
📝 提示词 #2 — 改进已有草稿
请帮我改进以下段落，要求：(1) 保持我的核心观点和事实不变 (2) 让语言更流畅自然，去掉生硬表达 (3) 增加一个具体例子让观点更有说服力 (4) 不要让改写后的文字"看起来像AI写的"。 [粘贴你的原文段落]
📝 提示词 #3 — 标题党生成器
为以下文章内容生成10个吸引点击的标题，要求：包含数字的3个、疑问句2个、"你不知道的"类型2个、直接结果导向3个。内容主题：[描述]。目标平台：[微信公众号/YouTube/小红书]。 生成后，请标注哪3个最可能获得高点击率，并说明原因。
📝 提示词 #4 — 营销邮件
你是一位专业邮件营销文案师。帮我写一封销售邮件，产品是[产品名称和描述]，目标客户是[描述]，核心痛点是[痛点]，我们的解决方案是[方案]。 要求：主题行吸引打开，正文150字以内，包含一个明确CTA，语气专业但不生硬，避免垃圾邮件关键词。
📝 提示词 #5 — 适配多平台改写
把下面这段内容分别改写成4个版本： 1. 微信公众号（800字，有深度，段落间有衔接） 2. 小红书（200字+5个emoji，轻松活泼，结尾加话题标签） 3. LinkedIn帖子（英文，300字，专业商务风格） 4. 推文/微博（50字以内，有冲击力） 原始内容：[粘贴内容]
📝 提示词 #6 — 产品描述
为以下产品写3版不同风格的描述，每版100字左右： 版本A：强调功能和规格（理性购买者） 版本B：强调情感价值和使用场景（感性购买者） 版本C：强调性价比和对比优势（价格敏感者） 产品：[产品名称和基本信息] 核心卖点：[3个卖点]

💻 编程 & 技术（8个）

💻 提示词 #7 — Debug助手
你是一位资深[语言]工程师。以下代码出现了错误，请帮我： 1. 找出所有可能的bug（至少说明3个潜在问题） 2. 解释每个bug产生的原因 3. 提供修复后的完整代码 4. 说明修改了哪些地方以及为什么 错误信息：[粘贴报错] 代码：[粘贴代码]
💻 提示词 #8 — 代码重构
请重构以下代码，要求： - 遵循[语言]最佳实践和设计模式 - 提高可读性（有意义的变量名、清晰的函数拆分） - 添加必要的注释（说明"为什么"而不是"是什么"） - 考虑边界条件和错误处理 - 保持原有功能100%不变 重构完成后，列出你做了哪些改动以及改动的理由。 [粘贴原始代码]
💻 提示词 #9 — 技术方案设计
我需要设计一个[系统/功能]，背景是：[描述需求和约束条件]。 请提供： 1. 2-3种可行的技术方案，说明各自的优缺点 2. 你最推荐哪种方案及理由 3. 推荐方案的核心数据结构/架构图（用文字描述） 4. 实现过程中可能遇到的主要技术挑战 5. 伪代码或代码框架（主要模块）
💻 提示词 #10 — SQL查询优化
以下SQL查询在生产环境中运行很慢（表有[数量]行数据）。请帮我： 1. 分析查询效率低的原因 2. 提供优化后的SQL版本 3. 建议添加哪些索引 4. 如果这个查询无法进一步优化，提供替代的查询策略 数据库：[MySQL/PostgreSQL/其他] 原始查询：[粘贴SQL]

📚 学习 & 研究（8个）

📚 提示词 #11 — 费曼学习法
请用"向10岁小孩解释"的方式讲解"[概念]"。要求： - 避免使用专业术语，如果必须用请立即解释 - 至少用2个日常生活中的类比 - 举一个具体的实际例子 - 最后用一句话总结核心思想 解释完后，给我出3道测试题，帮我验证自己是否真正理解了。
📚 提示词 #12 — 快速入门新领域
我想在2周内了解[领域/技能]的基础知识，每天可投入1-2小时。请为我制定： 1. 一个14天的学习路线图（每天学什么） 2. 最值得优先学习的5个核心概念 3. 3个免费/低成本的最佳学习资源 4. 2个可以检验学习成果的实践项目 5. 该领域最常见的5个误解
📚 提示词 #13 — 文献综述助手
我在研究"[研究话题]"，请帮我： 1. 总结这个领域的3-5个主要研究方向 2. 指出目前学界的主要争议点 3. 列出5个我应该了解的关键理论/模型 4. 推荐阅读顺序（从入门到进阶） 5. 指出该领域目前还没有解决的开放性问题 注意：如果你不确定某些信息，请明确说明"这需要查证"，不要编造引用。
📚 提示词 #14 — 备考复习
我正在备考[考试名称]，考试范围包括[列出章节/主题]。 请帮我： 1. 生成20道模拟题（选择题15道+简答题5道），覆盖以上所有章节 2. 每道选择题提供4个选项，标注正确答案和错误原因 3. 每道简答题提供参考答案要点 4. 指出哪些知识点是高频考点（标注⭐） 5. 给出3个记忆这些知识点的助记方法

💼 职场 & 商务（8个）

你是一位有15年经验的职业顾问和HR总监。请帮我优化以下简历，针对这个职位：[粘贴职位描述] 优化要求： 1. 量化每个成就（加入数字/百分比/规模） 2. 用职位描述中的关键词重写技能部分（ATS优化） 3. 调整每条经历的描述，突出与目标职位最相关的内容 4. 指出简历中的3个最大弱点和改进建议 我的简历：[粘贴简历]
💼 提示词 #16 — 会议总结
请将以下会议记录整理成标准格式，包括： - 会议基本信息（时间/参与者/主题） - 关键决策（用项目符号列出） - 行动事项（负责人 + 截止日期 + 具体任务） - 待解决问题（需要后续跟进的事项） - 下次会议议题建议 会议记录：[粘贴原始记录]
💼 提示词 #17 — 谈判准备
我即将进行[谈判类型：薪资/合同/商务合作]谈判，情况如下： 我方：[你的立场、底线、理想目标] 对方：[对方可能的立场、需求] 背景：[相关市场数据或情境] 请帮我： 1. 分析我方和对方的BATNA（最佳替代方案） 2. 列出5个可能的谈判策略 3. 准备10个关键问题用于探清对方底线 4. 为对方最可能提出的3个反对意见准备回应 5. 给出谈判开场白的建议措辞
💼 提示词 #18 — SWOT分析
请为以下业务/项目进行全面的SWOT分析：[描述业务或项目] 要求： - 每个维度列出5-7个具体、有洞察力的要点（不要泛泛而谈） - 基于SWOT提出3个战略建议（SO/ST/WO/WT策略各至少一个） - 指出最需要立即关注的1个威胁和1个机会 - 评估整体风险等级（高/中/低）并说明理由

📣 营销 & 社媒（7个）

📣 提示词 #19 — 小红书爆款文案

📣 提示词 #20 — YouTube视频脚本
帮我写一个[时长]分钟YouTube视频的完整脚本，话题：[话题]。 结构要求： - 前15秒：钩子（直接说出视频最大价值） - 1-2分钟：问题/痛点描述（引发共鸣） - 主体：[列出3-5个主要内容点] - B-roll提示：在脚本旁边标注[B-ROLL: 建议画面] - 结尾：总结+订阅引导 语气要求：口语化、有个性、不像念稿子
📣 提示词 #21 — 用户评价回复
帮我为以下用户评价写专业回复： 评价内容：[粘贴评价] 评价类型：[好评/差评/中评] 品牌语气：[正式专业/亲切温暖/活泼年轻] 回复要求： - 好评：表达真诚感谢，提炼用户的具体好评点 - 差评：承认问题、道歉、说明解决方案、邀请私下联系 - 不超过100字，不要模板腔 给我提供2个版本供选择。

🚀 进阶提示词技巧

🎯
技巧1：给AI设定角色和受众
相同问题，加上角色说明，输出质量大幅提升

❌ 弱版本："帮我写一封道歉邮件"
✅ 强版本："你是一位擅长危机公关的公司CEO。帮我写一封面向企业客户的道歉邮件，关于最近产品出现的数据延迟问题。语气要：承认错误但不引发恐慌，展现解决问题的决心，维护品牌信任。邮件100字以内。"

🔄
技巧2：要求AI自我审查
让AI检查自己的输出，错误率大幅降低

在任何重要请求的末尾加上："完成后，请检查你的回答：(1) 有没有事实性错误或不确定的信息？(2) 有没有遗漏关键点？(3) 是否真正回答了我的问题？如有问题请修正。"

📊
技巧3：要求多个版本对比
让AI提供3个选项，总比第一个好

不要让 AI 只生成一个结果。永远加上："给我3个不同方向的版本：第1个[某方向]，第2个[另一方向]，第3个[第三方向]。最后说明你认为哪个最好及原因。" 然后你从3个里选最好的，或者融合多个版本。

💡 使用提示词的核心原则

原则一：具体 > 模糊。"帮我写一封邮件"不如"帮我写一封100字的道歉邮件，给迟交报告的客户，语气要真诚但不卑微"。

原则二：给背景 > 直接提问。AI 不知道你的处境，多说一句"我是X，目前面临Y情况，我需要Z"，输出质量会显著提升。

原则三：迭代 > 重写。第一次结果不满意？不要重新开始，直接说"第3点太模糊，给我更具体的例子"或"重写第2段，语气更轻松一点"。

原则四：验证 > 盲信。对任何包含事实数据、引用来源的内容，用 Perplexity 或 Google 验证后再使用。
